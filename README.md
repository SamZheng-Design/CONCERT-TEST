# CONCERT TEST - Cardi B 演唱会项目测算系统

- **目标**: 演唱会项目财务测算与投资分成模型分析
- **技术栈**: Hono + TypeScript + TailwindCSS + Chart.js

## 核心功能

### 1. 票务区域设置
- 7个票区管理（内场 ¥2000-1580 / 看台 ¥1280-580）
- 场次编辑、添加/删除场地
- 饼图可视化收入分布

### 2. 成本结构
- 6大成本分类（艺人费用、差旅、制作、场地、营销、行政）
- 可展开查看明细并编辑
- 总成本约 ¥72,019,078

### 3. 赞助收入
- 文化局补贴 ¥10,000,000
- 指定赞助 ¥12,000,000
- 佣金率可调、净收入自动计算

### 4. 投资人分成模型（收入分成模式）
| 角色 | 投资额 | 说明 |
|------|--------|------|
| 优先级 | ¥2,000万 | 直接从票房收入分成，不承担费用 |
| 劣后级 | ¥3,000万 | 承担所有费用，获取剩余收入 |

**关键参数**:
- 收益倍数: 1.11x
- 退出目标: ¥2,220万
- 分成比例: 0-100% 可调

### 5. 压力测试
- 优先级退出分析图表
- 劣后级盈亏分析图表
- 售票率滑块（0%-100%）
- 退出平衡点标注

## 收入分成模式说明

```
优先级分成 = min(退出目标, 票房收入 × 分成比例)
劣后级收入 = 票房收入 - 优先级分成 + 赞助收入 - 总成本
```

- **优先级不承担费用**，直接从票房收入中分成
- 当分成比例为100%时，5%售票率对应的票房收入全部归优先级
- 优先级达到退出目标后封顶，超出部分归劣后级

## 数据模型
- **票务数据**: 场馆、票区、票价、容量
- **成本数据**: 6大分类、明细项目、单价×数量
- **投资数据**: 优先/劣后级金额、倍数、分成比例
- **存储**: 浏览器 LocalStorage 自动保存
